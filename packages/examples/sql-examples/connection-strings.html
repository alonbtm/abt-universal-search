PostgreSQL Connection String Formats

Standard URI Format RECOMMENDED
# Basic format postgresql://username:password@hostname:port/database # With SSL and additional parameters postgresql://user:pass@host:5432/dbname?sslmode=require&connect_timeout=10&application_name=MyApp # Example configurations postgresql://myuser:mypassword@localhost:5432/search_db postgresql://admin:secret123@db.example.com:5432/production_db?sslmode=require postgresql://readonly:readpass@replica.db.com:5432/analytics?sslmode=prefer&connect_timeout=30
Key-Value Pairs Format
# Space-separated key-value pairs host=localhost port=5432 dbname=search_db user=myuser password=mypassword sslmode=require # Multi-line format for readability host=localhost port=5432 dbname=search_db user=myuser password=mypassword sslmode=require connect_timeout=10 application_name=SearchApp
Environment Variable Pattern
# .env file POSTGRES_HOST=localhost POSTGRES_PORT=5432 POSTGRES_DATABASE=search_db POSTGRES_USER=myuser POSTGRES_PASSWORD=mypassword POSTGRES_SSL_MODE=require POSTGRES_CONNECTION_TIMEOUT=10 # Application code const connectionString = `postgresql://${process.env.POSTGRES_USER}:${process.env.POSTGRES_PASSWORD}@${process.env.POSTGRES_HOST}:${process.env.POSTGRES_PORT}/${process.env.POSTGRES_DATABASE}?sslmode=${process.env.POSTGRES_SSL_MODE}`; # Or using URL constructor for better security const dbUrl = new URL('postgresql://'); dbUrl.hostname = process.env.POSTGRES_HOST; dbUrl.port = process.env.POSTGRES_PORT; dbUrl.pathname = `/${process.env.POSTGRES_DATABASE}`; dbUrl.username = process.env.POSTGRES_USER; dbUrl.password = process.env.POSTGRES_PASSWORD; dbUrl.searchParams.set('sslmode', process.env.POSTGRES_SSL_MODE);

MySQL Connection String Formats

MySQL URI Format RECOMMENDED
# Basic format mysql://username:password@hostname:port/database # With SSL and charset mysql://user:pass@host:3306/dbname?ssl=true&charset=utf8mb4&timezone=%2B00%3A00 # Example configurations mysql://myuser:mypassword@localhost:3306/search_db mysql://admin:secret123@mysql.example.com:3306/production_db?ssl=true&charset=utf8mb4 mysql://app_user:app_pass@cluster.mysql.com:3306/app_db?ssl=true&acquireTimeout=60000
Traditional MySQL Format
# Traditional format (mysql2 specific) mysql2://user:password@host:port/database # With connection pool options mysql2://user:pass@host:3306/db?connectionLimit=10&acquireTimeout=60000&timeout=60000 # Example with SSL mysql2://myuser:mypass@mysql.server.com:3306/mydb?ssl=Amazon%20RDS&charset=utf8mb4
Environment Variable Pattern
# .env file MYSQL_HOST=localhost MYSQL_PORT=3306 MYSQL_DATABASE=search_db MYSQL_USER=myuser MYSQL_PASSWORD=mypassword MYSQL_SSL=true MYSQL_CHARSET=utf8mb4 MYSQL_CONNECTION_LIMIT=10 # Application code const connectionString = `mysql://${process.env.MYSQL_USER}:${process.env.MYSQL_PASSWORD}@${process.env.MYSQL_HOST}:${process.env.MYSQL_PORT}/${process.env.MYSQL_DATABASE}?ssl=${process.env.MYSQL_SSL}&charset=${process.env.MYSQL_CHARSET}`; # Configuration object approach (recommended for MySQL) const mysqlConfig = { host: process.env.MYSQL_HOST, port: parseInt(process.env.MYSQL_PORT), user: process.env.MYSQL_USER, password: process.env.MYSQL_PASSWORD, database: process.env.MYSQL_DATABASE, ssl: process.env.MYSQL_SSL === 'true', charset: process.env.MYSQL_CHARSET, connectionLimit: parseInt(process.env.MYSQL_CONNECTION_LIMIT) };

SQLite Connection String Formats

File Path Format MOST COMMON
# Absolute file path /path/to/database/search.db /home/user/data/app.sqlite /var/lib/myapp/database.sqlite3 # Relative file path ./data/search.db ../shared/database.sqlite search.db # Windows paths C:\data\search.db .\data\search.db
SQLite URI Format
# File URI format file:/path/to/database.db file:///absolute/path/to/database.db file://./relative/path/database.db # With parameters file:database.db?cache=shared&mode=rwc file:/path/to/db.sqlite?cache=private&mode=ro # Special databases :memory: # In-memory database "" # Temporary database file::memory: # Named in-memory database file:memdb1?mode=memory&cache=shared # Shared in-memory database
Environment Variable Pattern
# .env file SQLITE_DATABASE_PATH=./data/search.db SQLITE_DATABASE_MODE=rwc SQLITE_CACHE_MODE=shared SQLITE_TIMEOUT=5000 SQLITE_MEMORY_LIMIT=50000000 # Application code const databasePath = process.env.SQLITE_DATABASE_PATH || './default.db'; const connectionString = `file:${databasePath}?mode=${process.env.SQLITE_DATABASE_MODE}&cache=${process.env.SQLITE_CACHE_MODE}`; # Configuration object for better-sqlite3 const sqliteConfig = { path: process.env.SQLITE_DATABASE_PATH, options: { readonly: process.env.SQLITE_MODE === 'ro', fileMustExist: process.env.SQLITE_MUST_EXIST === 'true', timeout: parseInt(process.env.SQLITE_TIMEOUT), verbose: process.env.NODE_ENV === 'development' ? console.log : null } };

Security Best Practices

⚠️ NEVER DO THIS:

// DON'T: Hardcoded credentials in source code const connectionString = 'postgresql://admin:password123@prod-db.com:5432/production'; // DON'T: Credentials in version control const config = { database: 'mysql://root:admin123@localhost/app' }; // DON'T: Credentials in client-side code window.DB_CONNECTION = 'postgresql://user:pass@server/db'; // DON'T: Unencrypted storage of connection strings localStorage.setItem('db_connection', 'mysql://user:pass@server/db');

✅ SECURITY BEST PRACTICES:

// DO: Use environment variables const connectionString = process.env.DATABASE_URL; // DO: Use secure credential management const config = { host: process.env.DB_HOST, user: process.env.DB_USER, password: process.env.DB_PASSWORD, // From AWS Secrets Manager, HashiCorp Vault, etc. database: process.env.DB_NAME }; // DO: Validate connection string format function validateConnectionString(connectionString) { try { const url = new URL(connectionString); if (!['postgresql:', 'mysql:', 'file:'].includes(url.protocol)) { throw new Error('Invalid database protocol'); } return true; } catch (error) { throw new Error('Invalid connection string format'); } } // DO: Use proxy services for client-side applications const searchConfig = { dataSource: { type: 'sql', sql: { proxyEndpoint: 'https://api.myservice.com/search', // No direct DB access tableName: 'documents', searchFields: ['title', 'content'] } } };

Security Checklist:

  • Never hardcode credentials in source code
  • Use environment variables or secure secret management
  • Enable SSL/TLS for network database connections
  • Implement connection timeouts and retry logic
  • Use least-privilege database users
  • Validate and sanitize connection parameters
  • Use proxy services for client-side database access
  • Regularly rotate database passwords
  • Monitor and log database connection attempts
  • Use connection pooling to prevent connection exhaustion
